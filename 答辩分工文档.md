## A — 项目概述 + 前端 + 设计

### A 要讲的内容

#### 1. 项目背景与定位（1-2 分钟）

- 产品定位：极简个人健康管理 App
- 解决什么问题：日常健康追踪（饮食、运动、饮水、睡眠）一站式管理
- 目标用户：关注健康的年轻人，追求简洁不复杂

#### 2. 技术栈概览

- 前端：Vue 3 + TypeScript + Vite
- 后端：Spring Boot 3 + JPA + H2
- 移动端：Capacitor 7 打包 Android APK
- 通信方式：Axios HTTP 请求 + JSON 数据格式
- 认证：JWT Token

#### 3. 设计哲学（重点）

- **单页面单任务**：一个界面只做一件事（举例：饮水页只做喝水记录）
- **大面积留白**：用字号粗细代替边框（举例：首页卡片无边框设计）
- **莫兰迪绿** `#8FA89B` 作为唯一强调色，保持视觉统一

#### 4. 前端页面演示（过一遍核心页面）

|---|---|---|
| 启动页 | `/` | 淡入动画 Logo，1.2s 后跳转 |

#### 5. 前端关键技术点（答辩重点）

1. **Vue Router 懒加载**：

```typescript
// 按需加载页面，减少首屏体积
{ path: '/dashboard', component: () => import('../views/DashboardView.vue') }
```

2. **Axios 请求拦截器**：

```typescript
// 自动附加 JWT Token
client.interceptors.request.use(config => {
  const token = localStorage.getItem('accessToken')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 401 自动跳转登录
client.interceptors.response.use(
  res => res,
  err => {
    if (err.response?.status === 401) {
      router.push('/login')
    }
    return Promise.reject(err)
  }
)
```

3. **SVG 圆环进度**（纯 CSS/SVG 实现，无第三方图表库）：

```html

<circle cx="44" cy="44" r="38" stroke="#8FA89B" stroke-width="8" fill="none"
  :stroke-dasharray="239"
  :stroke-dashoffset="239 - 239 * (healthScore / 100)"
  stroke-linecap="round" transform="rotate(-90 44 44)"/>
```

4. **FAB 悬浮按钮**：点开弹出"记饮食、开始跑步、记饮水"快捷入口

#### A 预期问题与回答

| 可能的问题 | 参考回答 |
|---|---|
| **为什么用 Vue 3 而不是 React？** | Vue 3 上手门槛低，Composition API 适合中小型项目，Vite 开发构建速度快 |
| **为什么选择 Capacitor 而非 React Native / Flutter？** | 项目已经是 Vue 开发，Capacitor 可以直接把 Web App 打包成 Android APK，零成本迁移；RN/Flutter 需要重写全部 UI |
| **没有第三方图表库怎么画图？** | 直接用 SVG 的 `circle` + `stroke-dasharray` / `stroke-dashoffset` 实现圆环进度，无需引入 echarts 等重型库 |
| **页面多，性能如何保证？** | 路由懒加载，按需加载页面组件，首屏只加载启动页 |
| **设计上有什么亮点？** | 每页只有一个核心操作（单页面单任务）；大面积留白；莫兰迪色调统一视觉 |

## B — 后端 + 数据库 + 部署

### B 要讲的内容

#### 1. 后端架构（Spring Boot 三层架构）

```
Controller (接收请求) → Service (业务逻辑) → Repository (数据访问)
                                       ↕
                                  Entity (数据模型)
```

- **Controller 层**：10 个 REST 控制器，接收 HTTP 请求，返回统一 JSON 格式
- **Service 层**：10 个业务服务，处理具体逻辑
- **Repository 层**：JPA 继承 `JpaRepository`，自动实现 CRUD
- **Entity 层**：JPA 注解映射数据库表

#### 2. API 接口一览

| 端点 | 方法 | 功能 | 认证 |
|---|---|---|---|
| `/api/auth/login/password` | POST | 密码登录 | ❌ |
| `/api/auth/register` | POST | 注册 | ❌ |
| `/api/auth/refresh` | POST | 刷新 Token | ❌ |
| `/api/dashboard` | GET | 今日看板数据 | ✅ |
| `/api/diet/**` | GET/POST | 饮食记录 | ✅ |
| `/api/water` | GET/POST/DELETE | 饮水追踪 | ✅ |

#### 3. 数据库设计（核心实体）

```
User (用户)
├── id, phone, passwordHash
├── name, gender, birthDate
├── heightCm, weightKg, activityLevel
├── profileComplete
│
├── HealthGoal (健康目标) — 一对一
│   └── dailySteps, dailyWaterMl, dailySleepH
│

│   └── steps, waterMl, sleepHours, restingHr

```

#### 4. 健康指数算法（核心业务逻辑）

```java
public static int compute(DailyStats stats, HealthGoal goal) {
    double stepScore = min(steps / goal.dailySteps, 1.0) × 30;     // 步数 30分
    double waterScore = min(waterMl / goal.dailyWaterMl, 1.0) × 20; // 饮水 20分
    double sleepScore = min(sleepHours / goal.dailySleepH, 1.0) × 25; // 睡眠 25分
    // 静息心率 25分 — 分段打分
    if (hr < 50)  hrScore = 10;
    if (hr < 60)  hrScore = 20;
    if (hr <= 80) hrScore = 25;  // 最佳区间满分
    if (hr <= 90) hrScore = 18;
    else          hrScore = 10;
    // 总分 0-100
}
```

#### 5. 安全机制

- **Spring Security** 配置：`/api/auth/**` 无需认证，其余需要 JWT
- **JWT 流程**：

```
登录 → 后端生成 accessToken(15分钟过期) + refreshToken(7天过期)
     → 前端存 localStorage
     → 每次请求通过 Authorization: Bearer xxx 携带
     → JwtAuthenticationFilter 拦截验证
     → 401 时前端用 refreshToken 换新 accessToken
```

- **BCrypt 密码加密**：不存明文密码
- **无状态会话**（`STATELESS`）：服务器不保存 Session，适合移动端

#### 6. H2 数据库

- `jdbc:h2:file:./data/minimalhealth` — 文件模式持久化（重启不丢数据）
- `MODE=MySQL` — 语法兼容 MySQL，方便未来迁移
- `ddl-auto: update` — JPA 自动建表/更新表结构
- 提供 `/h2-console` Web 管理界面，可在线查看数据

#### 7. Capacitor 打包 Android APK

```
npm run build         → 构建前端静态文件
npx cap sync android  → 同步到 Android 项目
cd android
./gradlew assembleDebug → 编译 APK
```

生成 APK 路径：`android/app/build/outputs/apk/debug/app-debug.apk`

#### B 预期问题与回答

| 可能的问题 | 参考回答 |
|---|---|
| **为什么用 Spring Boot 而不是其他框架？** | Spring Boot 生态成熟，JPA 自动建表减少 SQL 编写，起步依赖一键集成，适合快速开发 REST API |
| **JWT 和 Session 有什么区别？为什么选 JWT？** | Session 需要服务端存储，移动端 App 不适合做 Cookie Session；JWT 无状态，移动端存 Token 即可，扩展性好 |
| **H2 能用于生产吗？** | H2 是轻量级嵌入式数据库，适合开发/演示/单机小规模使用。生产上量后替换成 MySQL 只需改一行配置（因为已设 `MODE=MySQL`） |
| **密码怎么存储的？安全吗？** | BCrypt 加密，不可逆哈希，每次加密结果不同（自动加盐），目前最安全的密码哈希算法之一 |
| **后端异常怎么处理的？** | 全局异常处理器 `GlobalExceptionHandler`，统一返回 `{ code, message, data }` 格式，不会把异常栈泄露给前端 |
| **APK 怎么打包的？** | Capacitor 把 Vue 构建产物 `dist/` 同步到 Android 原生壳工程，然后通过 Gradle 编译成 APK，本质是 WebView 加载前端页面 |
| **怎么和前端通信的？** | HTTP REST API，JSON 格式，Axios 库发请求，前端和后端完全分离 |
